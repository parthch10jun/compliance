## 📁 Data Structure for Each Framework

Each CBUAE framework will include:

### 1. Program Template
```typescript
{
  id: 'tpl-cbuae-[framework-id]',
  name: 'CBUAE [Framework Name]',
  description: 'Brief description',
  longDescription: 'Detailed description with scope and applicability',
  tags: ['CBUAE', 'Regulator', 'UAE', '[Category]', 'Banking'],
  framework: 'CBUAE [Short Name]',
  version: '2024',
  category: 'Regulatory',
  requirementCount: XX,
  obligationCount: XX,
  controlCount: XX,
  publisher: 'Central Bank of UAE',
  effectiveDate: 'YYYY-MM-DD',
  lastUpdated: '2024-XX-XX',
  popularity: 0,
  keyFeatures: [...],
  applicableTo: ['Banks', 'Financial Institutions', 'UAE'],
}
```

### 2. Requirements (Continuous-state compliance)
- Mapped to specific articles/sections of CBUAE regulations
- Tagged with framework, category, risk rating
- Linked to controls

### 3. Obligations (Time-bound, event-driven)
- Reporting requirements (quarterly, annual)
- Notification requirements (incident reporting)
- Submission deadlines
- Regulatory filings

### 4. Controls
- Reusable across multiple CBUAE frameworks
- Tagged for easy filtering
- Linked to tests and evidence
